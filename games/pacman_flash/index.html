Objective:

  • The objective of the game is to eat all the dots adn power-ups.
  • Enemies try to kill the PacMan. Different colored enemies have different behaviour.
  • Eating a power-up will dissable all enemies, and makes them go slowly in random.
  • Enemies can be eaten when they are dissabled state, for extra points. Enemies then goto their home and reset.
  • If the pacman looses all the lives, hen the game is over.
  • There is a wrap zone in the middle of the map on left and right.


Objective:

  • Move the pac-man in all four directions using the respective arrow keys.


Logic Design and Implementation:

  • Create the level bitmap. It contains all elements (walls, dots, powerups). It also contans information about junctions.
  • Create movement logic, so that it will check the level bitmap for junctions and changes direction if possible in the next junction.
  • Create movement logic for enemies. At every junction, enemies determine in which direction the player is and acts accordingly.
  • Create four different logics for four different enemies.
    • Red enemy is more aggressive. it will try to catch pacman most of the time.
    • Blue enemy sometimes changes direction. Tt only checks half of the level for player.
    • Pink enemy checks radius around it for player. Tt gives up chase after sometime and resumes.
    • Orange tries to stay away from the player. It attacks only when player is very close to it.
  • Create variables to hold the score and lives of pac-man.
  • Create logics to determine when the game is over and display it on the screen.
  • Integrate audio into the game, and also a mute button to mute the audio.
